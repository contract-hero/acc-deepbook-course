What you'll build

A React dashboard that shows live market stats for every pool the local sandbox deploys. Per-pool: mid price, bid/ask spread, depth within ±1%, 24h volume, last fill, and a sparkline. The reference implementation passes a 24-test equivalence suite — your version just has to match.

  • Manifest bootstrap. Parse the sandbox's localnet.json into a typed PoolDescriptor[]; share the validator between Node and browser.
  • RPC foundation. Talk to Sui JSON-RPC directly — no SDK shortcuts. Build the request shapes from chain-shape.md, not from training memory.
  • Order-book decoding. Walk a Versioned<PoolInner> wrapping a Book with BigVector<Order> sides; decode order_id into price.
  • Fill aggregation. Paginate OrderInfo events into 24h volume + last-fill state without an indexer.
  • Market stats. Derive mid, spread, depth-within-1% from the decoded book; render the sparkline from your event stream.
  • The verification gate. A 24-test vitest suite the reference passes; your code doesn't have to look like the reference, it just has to behave like it.

How a lesson runs

ACC handles the setup. You handle the code (or watch — your call). Every section gates on a real test run before advancing.

Quick start

Three commands. The auto-bootstrap takes ~5-10 min on first run while it pulls Docker images and brings the sandbox up; after that, lesson start is instant.

1

Install the framework + this course

From any Claude Code session. The course requires agentic-community-college ≥ 0.2.0.

/plugin marketplace add contract-hero/plugin-marketplace
/plugin install agentic-community-college@contract-hero
/plugin install acc-deepbook-course@contract-hero
2

Make sure Docker is running

ACC will run pnpm deploy-all --quick against your local Docker. Without it, the sandbox-faucet probe blocks at preflight.

3

Start the lesson

ACC prompts for a workspace location on first run (default ~/workspace), auto-clones the sandbox, runs pnpm install, and brings up the stack — all as preflight remediations.

/acc-deepbook-course:start
Trust note. Auto-bootstrap runs git clone https://github.com/MystenLabs/deepbook-sandbox and pnpm install inside the cloned repo's sandbox/ subdir. The URL is auditable here in the public plugin manifest; review the upstream before approving the remediation if you don't trust it. Cold path is ~500 MB Docker images + ~100 MB git/install, so first run isn't great on cellular.
Want the sandbox somewhere other than ~/workspace/deepbook-sandbox?

Hand-edit ~/.acc/config.json. The plugin key in course_paths is whatever appears for this course in ~/.claude/plugins/installed_plugins.json — @contract-hero if you installed from the marketplace, @local for dev work.

{
  "workspace_root": "~/workspace",
  "course_paths": {
    "acc-deepbook-course@contract-hero": {
      "sandbox": "~/wherever/deepbook-sandbox"
    }
  }
}

A /acc:settings CLI for editing this without touching JSON is on ACC's roadmap. Until then, hand-edit, or call configureWorkspace from any ACC session.
