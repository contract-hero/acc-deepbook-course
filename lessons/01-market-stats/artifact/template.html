1Manifest bootstrap

Read localnet.json from disk (Node) or via fetch (browser). Validate. Surface two things: deepbookPackageId and a typed PoolDescriptor[].

Key moment: Keep the validator pure. fs.readFile goes in the Node wrapper, not in the validator — otherwise the browser bundle breaks the moment you import it.

2RPC foundation — sui_getObject + descent

The pool's state lives on chain inside a Versioned<PoolInner> wrapper. To read it you call sui_getObject, then walk the JSON down a specific field path. Get the path wrong and every later lookup returns undefined.

Key moment: content.fields.value.fields. The value.fields hop is the Versioned wrapper — easy to miss because the SDK normally hides it. No compile-time check protects you here; the tests in chainShape.test.ts are your safety net.

3Order book traversal + order_id decode

BigVector<Order> stores its elements in dynamic-field slices. To enumerate them you list the slices via suix_getDynamicFields, then read each slice's content with sui_getObject. Each Order carries an order_id: u128 that encodes price + side + sequence.

Key moment: The Order→OrderInfo-event join. Bid prices come out right if you prefer the event's price field; if you bit-decode them you get nonsense for half the order book.

4Pure market-stats math

With the order book decoded as two sorted { price, qty }[] arrays, the rest is just functions. Mid price, spread, depth band — pure inputs, pure outputs, no I/O.

Key moment: Anchor the depth band on each side's best price, not on mid. The math looks symmetric but it isn't — and the divergence shows up exactly when you need the metric to be most honest.

5Fills pagination + 24h aggregation

The order book is a snapshot. To compute last price, 24h volume, and the sparkline, you need recent trades — DeepBook emits OrderInfo events for every placement, with embedded fills under fills[].

Key moment: Both stop conditions. Drop (1) and a quiet sandbox makes you fetch forever; drop (2) and a busy minute wedges the pager on a single page.

6App wiring — graceful per-pool failures

Mount the React app. Load the manifest once. Fan out per-pool fetches every 20 seconds. One transient pool error must not blank the whole dashboard.

Key moment: Promise.allSettled + sentinelCard(). The PoolCard component renders — for any undefined field, so a sentinel card is just a card with every stat undefined. The user sees clearly-degraded data instead of a blank screen.
